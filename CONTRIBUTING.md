# Contributing

## How it works

This repo uses [Release Please](https://github.com/googleapis/release-please) to automate versioning and releases. **Do not manually edit `version` or `appVersion` in `Chart.yaml`, and do not manually edit the charts table in `README.md`** — these are managed automatically.

### Commit message convention

All commits must follow [Conventional Commits](https://www.conventionalcommits.org/). Release Please reads commit history to determine what version bump to apply and what goes in the changelog.

| Prefix | Effect | Example |
|--------|--------|---------|
| `feat:` | minor bump | `feat(traccar): add persistence support` |
| `fix:` | patch bump | `fix(esphome): correct service port` |
| `chore:` | patch bump | `chore(manyfold): update default values` |
| `deps:` | patch bump | `deps(manyfold): update postgresql subchart` |
| `feat!:` or `BREAKING CHANGE:` | major bump | `feat!(manyfold)!: redesign values schema` |

The scope (e.g. `feat(traccar):`) should be the chart name. This is what Release Please uses to attribute commits to a specific chart's release.

### Workflow

1. Make your changes to a chart under `charts/<name>/`
2. Open a PR against `main` — CI will run `helm lint` and `kubeconform` validation
3. Merge the PR with a conventional commit message
4. Release Please will open (or update) a release PR for that chart, bumping `Chart.yaml` version and updating `CHANGELOG.md`
5. When the release PR is merged, CI will automatically:
   - Package and publish the chart to the Helm repo and GHCR
   - Update the Helm index on `gh-pages`
   - Regenerate the charts table in `README.md` on both `main` and `gh-pages`

### Adding a new chart

1. Create `charts/<name>/` following the structure of an existing chart
2. Add the chart to `release-please-config.json` under `packages`:
   ```json
   "charts/<name>": {
     "component": "<name>",
     "paths": ["charts/<name>"]
   }
   ```
3. Open a PR — Release Please will pick it up on merge

### What not to touch

- `Chart.yaml` `version` field — bumped by Release Please
- `README.md` charts table — regenerated by CI on release
- `gh-pages` branch — managed entirely by CI
- `.release-please-manifest.json` — managed by Release Please
